System

Die Architektur ist absichtlich schlicht.

Die Daten liegen als Markdown im Repository. Das Frontend liest nur. Die spätere Logik soll flüchtig bleiben: serverlos, klein und austauschbar.

1. GitHub

Das Regal

Alles Wichtige lebt direkt im Repository: Inbox-Einträge, Bibliothek, Verlauf und Versionen.

2. Astro

Die Lesefläche

Das Frontend ist darauf optimiert, Dinge schnell wiederzufinden, lesbar zu machen und nicht im Weg zu stehen.

3. Vercel

Flüchtige Dienste

Später kann Vercel serverlos Inbox-Eingänge annehmen oder einen Kurationslauf anstossen, ohne selbst dauerhafter Speicher zu werden.

Späterer Agent

Der geplante Lauf in einem Satz

  • leseneinen rohen Inbox-Eintrag aus GitHub holen
  • formeninhaltlich klären, Titel und Tags vorschlagen
  • einordnenals Bibliothekseintrag zurück ins Repo schreiben
  • markierenden ursprünglichen Inbox-Zettel als verarbeitet kennzeichnen

GitHub ist das Regal. Astro ist nur die ruhige Leseschicht darüber.
